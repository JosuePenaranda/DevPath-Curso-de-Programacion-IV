TEMA 4: Elementos del Back-end

Spring Boot, Thymeleaf, HTTP, Cookies, Sesiones y más

Cliente vs Servidor

El cliente es quien inicia la comunicación (navegador, app móvil), enviando solicitudes. El servidor las recibe, las procesa y devuelve una respuesta. Es una arquitectura de solicitud-respuesta donde cada uno tiene un rol definido.

Método GET / POST
  • GETSolicita datos; parámetros en la URL, no modifica nada.
  • POSTEnvía datos al servidor; parámetros en el cuerpo, más seguro.
HTTP Response

Es la respuesta que el servidor envía al cliente. Contiene: un status code (200 OK, 404 Not Found, 500 Error), headers (tipo de contenido, cookies, etc.) y el body (HTML, JSON, etc.).

HTTP Request

Es la solicitud que el cliente envía al servidor. Contiene: el método (GET, POST…), la URL, los headers (idioma, tipo de contenido) y opcionalmente un body (en POST).

Spring Boot

Framework de Java que simplifica la creación de aplicaciones web. Elimina configuración manual gracias al autoconfigurado, incluye un servidor embebido (Tomcat) y permite levantar una app con muy poco código usando anotaciones.

Inyección de dependencias (@Autowired)

Permite que Spring cree y gestione objetos automáticamente. En lugar de hacer new MiServicio(), se declara con @Autowired y Spring inyecta la instancia. Reduce el acoplamiento y facilita el testing.

Controladores (@Controller / @RestController)

@Controller maneja peticiones y retorna vistas (HTML con Thymeleaf). @RestController retorna datos directamente (JSON/XML), ideal para APIs. La diferencia clave es que @RestController incluye @ResponseBody automáticamente.

Cookies

Pequeños archivos de texto que el servidor guarda en el navegador del cliente. Se usan para recordar información entre sesiones (preferencias, login). Se envían automáticamente en cada request.

Sesiones

Mecanismo del servidor para guardar información del usuario durante su visita. A diferencia de las cookies (cliente), los datos se guardan en el servidor. Spring usa HttpSession y cada usuario tiene un ID de sesión único guardado en una cookie.
