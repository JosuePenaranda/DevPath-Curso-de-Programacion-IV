Eventos de Mouse
click Mouse

Se dispara cuando el usuario presiona y suelta el botón principal del ratón sobre un elemento. Es el evento más usado para detectar interacciones del usuario, como pulsar botones o enlaces.

dblclick Mouse

Se activa cuando el usuario hace doble clic sobre un elemento. Se usa para acciones que requieren confirmación rápida, como abrir un archivo o editar un campo de texto.

mousedown Mouse

Se dispara en el momento exacto en que el botón del ratón es presionado, antes de soltarlo. Útil para iniciar acciones de arrastre (drag) o detectar el inicio de una pulsación.

mouseup Mouse

Se activa cuando el usuario suelta el botón del ratón. Complementa a mousedown y se usa para finalizar acciones de arrastre o detectar el fin de una pulsación.

mousemove Mouse

Se dispara continuamente mientras el cursor del ratón se mueve sobre un elemento. Se usa para rastrear la posición del mouse, crear efectos visuales o implementar funcionalidades de arrastre.

mouseenter Mouse

Se activa cuando el cursor entra al área de un elemento. A diferencia de mouseover, no se propaga a los elementos hijos, por lo que es más preciso para detectar la entrada al elemento específico.

mouseleave Mouse

Se dispara cuando el cursor sale del área de un elemento. Al igual que mouseenter, no se propaga a hijos. Se usa para ocultar tooltips o revertir efectos visuales al salir del elemento.

mouseover Mouse

Se activa cuando el cursor entra al elemento o a cualquiera de sus hijos. Se propaga (bubbling), lo que lo hace útil para detectar hover en elementos con contenido anidado.

mouseout Mouse

Se dispara cuando el cursor sale del elemento o de alguno de sus hijos. Se propaga igual que mouseover. Se usa para detectar cuando el usuario deja de interactuar con un área y sus elementos internos.

contextmenu Mouse

Se activa cuando el usuario hace clic derecho sobre un elemento, antes de que aparezca el menú contextual del navegador. Se usa para mostrar menús personalizados o bloquear el menú por defecto.

Eventos de Teclado
keydown Teclado

Se dispara cuando el usuario presiona una tecla del teclado. Se activa antes de que el carácter aparezca en el campo. Útil para detectar atajos de teclado o validar entradas en tiempo real.

keyup Teclado

Se activa cuando el usuario suelta una tecla. Se usa para ejecutar acciones después de que el usuario termina de escribir un carácter, como búsquedas en tiempo real o validaciones de formulario.

Eventos de Formulario
blur Formulario

Se dispara cuando un elemento pierde el foco, es decir, cuando el usuario hace clic fuera de él o navega con Tab a otro campo. Se usa para validar datos al salir de un campo de formulario.

focus Formulario

Se activa cuando un elemento recibe el foco, ya sea por clic o por navegación con teclado. Se usa para resaltar campos activos, mostrar sugerencias o limpiar mensajes de error.

change Formulario

Se dispara cuando el valor de un elemento de formulario (input, select, textarea) cambia y el elemento pierde el foco. Se usa para reaccionar a cambios en selects, checkboxes o campos de texto.

select Formulario

Se activa cuando el usuario selecciona texto dentro de un campo de texto o textarea. Se usa para detectar qué texto fue seleccionado y realizar acciones como copiar, resaltar o mostrar opciones de formato.

Eventos de Ventana
load Ventana

Se dispara cuando un recurso y todos sus recursos dependientes (imágenes, scripts, estilos) han terminado de cargarse. Se usa en window para ejecutar código una vez que la página está completamente lista.

DOMContentLoaded (en document) Ventana

Se activa cuando el HTML ha sido completamente cargado y analizado, sin esperar imágenes ni estilos externos. Es más rápido que load y se usa para manipular el DOM tan pronto como esté disponible.

resize Ventana

Se dispara cuando el tamaño de la ventana del navegador cambia. Se usa para adaptar el diseño dinámicamente, recalcular posiciones de elementos o ajustar gráficos y canvas.

scroll Ventana

Se activa cuando el usuario desplaza el contenido de un elemento o de la página. Se usa para implementar efectos de scroll infinito, animaciones al hacer scroll o mostrar/ocultar elementos según la posición.

beforeunload Ventana

Se dispara justo antes de que el usuario abandone la página (cierre la pestaña, navegue a otra URL o recargue). Se usa para mostrar un mensaje de confirmación cuando hay cambios sin guardar.

unload Ventana

Se activa cuando la página está siendo descargada, es decir, cuando el usuario navega a otra página o cierra la pestaña. Se usa para limpiar recursos, cancelar peticiones pendientes o guardar datos antes de salir.
